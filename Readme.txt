Программа-компрессор данных version 1.0
=======================================


Возможности программы
---------------------

- Сжатие данных любого формата без потерь при помощи алгоритма Хаффмана с возможностью шифрования
  данных пользовательским паролем;

- Разжатие предварительно сжатых данных.

При этом обе упомянутые операции (сжатие и расжатие) выполняются только с помощью данной программы.


Основной алгоритм программы для целей сжатия
-------------------------------------------- 

- Открытие сжимаемого файла и создание таблицы Хаффмана;

- Проверка целостности данных при помощи значения хеш-суммы;

- Создание сжатого файла с использовнием таблицы;

- Шифрование полученных данных(в случае если пользователь выбрал использование пароля).


Основной алгоритм программы для целей разжатия
----------------------------------------------

- Открытие сжатого файла и проверка на наличие пароля. При наличии пароля производится расшифровка данных.

- Декодирование данных и создание разжатого файла. 

Структура создаваемого файла
----------------------------
- 1 байт – индикатор того, что файл зашифрован и может быть открыт только с помощью пароля;
- 4 байта – контрольная сумма сжимаемого файла;
- 4 байта – длина имени файла;
- Столько байт, сколько символов в имени файла - само имя файла
- «Таблица» Хаффмана, которая состоит из 1 байта, в котором записана длина кода в битах, и самого кода, размер которого может варьироваться от 1 до 32 байт. Каждый номер «строки» соответствует номеру элемента таблицы ASCII.
- Сам сжатый файл

Инструкция пользователя 
-----------------------
 
В консоль пользователь должен вводить указаные ниже ключи:

- Ключ -c для сжатия файла.За вводом данного ключа следует ввод пути и имени сжимаемого файла, а также пути и имени
  файла, создаваемого в результате выполнения операции сжатия. 
  Если путь и имя сжатого файла не указаны, то сам файл будет создан в директории, а в качестве имени будет указано
  имя исходного файла с разрешением .cmp.

- Ключ -cp практически аналогичен ключу -c, но добавляет возможность создания пароля к сжатому файлу.
  При этом пароль не должен состоять более чем из 32 символов.

- Ключ -d для разжатия файла,предварительно сжатого программой. Здесь также указываются путь и имя разжимаемого файла. 

- Ключ -dp аналогичен ключу -d, но правильное декодирование* возможно только с паролем, который пользователь ввел при сжатии
  файла.

*Прим. При попытке пользователя выполнить расжатие зашифрованного паролем файла с использованием ключа -d или посредством
       ввода неправильного пароля расжатие файла будет выполнено неправильно.  

Минимальные системные требования
--------------------------------

- Windows 7/8/8.1/10 (32/64 bit)
